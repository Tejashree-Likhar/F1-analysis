# 🏎️ F1 Insights

A Formula 1 analytics dashboard exploring 25 seasons of real race data
(2000–2024), built with Streamlit and Plotly.

## What's inside

- **`src/data_prep.py`** – cleans the raw race dataset, fixes a couple of
  real data quirks (see below), and attaches driver/constructor/circuit
  names.
- **`src/train_model.py`** – trains two GradientBoosting models (podium
  classifier + points-per-race regressor) on the cleaned data. Not currently
  wired into the app, but kept here if you want to build predictive features
  later.
- **`src/theme.py`** – F1-inspired color palette & CSS for the app.
- **`app.py`** – the Streamlit dashboard, with 4 pages:
  - 🏠 Overview — headline stats
  - 📊 Historical Explorer — win/podium leaders, grid-vs-finish, wet vs dry,
    circuit factors
  - 📈 **Driver Timelines** — every driver's championship-points progression
    across a season, Grand Prix by Grand Prix; click a driver's name in the
    chart legend to isolate their line, or pick them from the dropdown for a
    race-by-race win/podium/points breakdown
  - ⚔️ Driver Comparison — head-to-head career stats

## Data

- **`data/race_data_raw.csv`** — the dataset you provided (per-race entries,
  2000–2024, Ergast-style schema).
- **`data/drivers.csv`, `constructors.csv`, `circuits.csv`, `races.csv`** —
  reference tables (from an open-source Ergast-schema mirror) used only to
  turn numeric IDs into readable names.
- **`data/race_data_clean.csv`** — generated by `data_prep.py`; this is what
  the app and models actually use.

### A data quirk worth knowing about

The raw `points` column is **not** the points scored at that specific Grand
Prix — it's identical to `points_driver_champ`, i.e. the driver's cumulative
championship total through that race (verified: the two columns match on
every single row). `data_prep.py` derives a proper **`race_points`** column
(the race-to-race increase in that cumulative total) for anything that
needs "how many points did they score at this race" — win/podium leaders,
the Driver Timelines charts, grid-vs-finish, wet-vs-dry, etc. Career/season
totals use the max of `points_driver_champ` per season instead of summing
the (cumulative) `points` column.

## Setup in VS Code

1. **Open the folder** in VS Code (`File → Open Folder…`, select this
   project's folder).

2. **Create a virtual environment** (Terminal → New Terminal, then):

   ```bash
   python3 -m venv .venv
   ```

   Activate it:
   - macOS/Linux: `source .venv/bin/activate`
   - Windows (PowerShell): `.venv\Scripts\Activate.ps1`

3. **Install dependencies:**

   ```bash
   pip install -r requirements.txt
   ```

4. **Run the app:**

   ```bash
   streamlit run app.py
   ```

5. **Generate the data/models:**

   ```bash
   python src/data_prep.py
   python src/train_model.py
   ```
